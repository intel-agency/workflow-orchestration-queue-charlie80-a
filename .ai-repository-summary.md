# AI Repository Summary

## Repository Purpose

**workflow-orchestration-queue-charlie80-a** is a GitHub Actions-based AI orchestration system that automates workflow management through intelligent agent delegation. The system uses the `opencode` CLI runtime with ZhipuAI GLM models to process GitHub events (issues, PR comments, reviews) and delegate work to specialized sub-agents.

### Primary Components

1. **Orchestrator Agent** - Coordinates specialist agents, never writes code directly
2. **Specialist Agents** - Domain-specific agents in `.opencode/agents/` (developer, code-reviewer, devops-engineer, etc.)
3. **GitHub Actions Workflows** - Automated CI/CD and agent orchestration
4. **Devcontainer Environment** - Reproducible development environment with all tools pre-installed

## Development Workflow

### 1. Environment Setup
1. Open repository in devcontainer (automatically pulls prebuilt image from GHCR)
2. Devcontainer auto-starts `opencode serve` on port 4096
3. Environment includes all tools: .NET SDK, Node.js, Bun, uv, opencode CLI, GitHub CLI

### 2. Making Changes
1. Create feature branch from `main`
2. Make minimal, targeted changes
3. Run validation: `pwsh -NoProfile -File ./scripts/validate.ps1 -All`
4. Ensure all tests pass before committing

### 3. Commit Guidelines
- Keep changes minimal and focused
- Follow conventional commit format: `feat:`, `fix:`, `docs:`, `chore:`
- Pin all GitHub Actions to full SHA (not tags)
- Never commit secrets or tokens

### 4. CI/CD Pipeline
- **validate** workflow runs on PR: lint, scan, test, devcontainer build
- **publish-docker** workflow publishes base image to GHCR
- **prebuild-devcontainer** workflow layers Features on base image
- **orchestrator-agent** workflow triggers AI agent orchestration on GitHub events

## Important Notes

### Agent Orchestration
- Orchestrator delegates to specialists via `task` tool
- Never writes code directly
- Maximum delegation depth: 2 levels

### Security
- All GitHub Actions must be SHA-pinned (no tags/branches)
- Secrets stored in repository Settings → Secrets
- Never commit `.env` files or credentials

### Template Repository
This is a GitHub template repository. New instances are created via automation scripts in `nam20485/workflow-launch2`. Template placeholders (`workflow-orchestration-queue-charlie80-a`, `intel-agency`) are replaced during creation.
